# Markdown 轉 HTML 工具

一個功能完整的 Markdown 編輯器，支援即時預覽和原生 Markdown 轉 HTML 轉換。

## 功能特色

### 編輯器功能
- ✨ **即時預覽**：左側編輯，右側即時顯示 HTML 預覽
- 📝 **行號顯示**：編輯器左側顯示行號，與編輯器內容同步滾動
- 🎯 **游標追蹤**：即時顯示當前游標位置（行數和列數）
- 🔄 **滾動同步**：編輯器和預覽區滾動位置自動同步
- 📋 **一鍵複製**：點擊複製按鈕快速複製 Markdown 內容，並顯示 Toast 提示
- 📏 **可調整大小**：左右兩個區塊可以拖動中間分隔線調整寬度（限制在 20%-80%）
- 📚 **語法指南**：內建 Markdown 語法指南，可展開/收起查看常用語法範例

### 智能編輯功能
- ⌨️ **Tab 鍵插入空格**：按 Tab 鍵自動插入 2 個空格（符合 Markdown 規範）
- 🎯 **自動滾動定位**：編輯器游標移動時，預覽區自動滾動到對應行

### 技術特色
- 🎨 **原生實現**：使用原生 JavaScript 實現 Markdown 轉換，無需額外套件
- 🎨 **現代化 UI**：使用 Tailwind CSS 4 構建美觀的用戶界面
- 📱 **響應式設計**：適配不同螢幕尺寸

## 技術棧

- React 19
- Vite
- Tailwind CSS 4
- 原生 JavaScript（無 Markdown 解析套件）

## 開發工具

本專案使用以下 AI 輔助開發工具：

- **Cursor**：AI 驅動的程式碼編輯器
- **ChatGPT**：元件邏輯生成
- **Auto (Cursor AI)**：AI 程式碼助手，用於：
  - 程式碼生成與重構
  - 功能實現與除錯
  - 文件撰寫與優化
  - 技術問題解決

## 安裝與運行

```bash
# 安裝依賴
npm install

# 開發模式
npm run dev

# 建置
npm run build

# 預覽建置結果
npm run preview
```

## 專案結構

.
├── public/
│   └── assets/               # 靜態資源（圖片等）
├── docs/                     # Build 輸出目錄（GitHub Pages）

│   ├── components/
│   │   ├── MarkdownEditor.jsx    # Markdown 編輯器元件

│   │   └── Toast.jsx             # Toast 提示元件
│   ├── utils/
│   │   └── markdownParser.js     # Markdown 轉 HTML 解析器（原生實現）
│   ├── config/

│   ├── App.jsx                   # 主應用程式

├── vite.config.js               # Vite 配置（含 GitHub Pages base path）
└── package.json                 # 專案依賴配置

## AI 生成代碼的優缺點

### 優點 ✅

1. **開發效率提升**
   - 快速生成基礎程式碼結構，減少重複性工作
   - 自動完成常見功能實現，加速開發流程
   - 提供多種解決方案參考，擴展思路

2. **學習與探索**
   - 快速學習新技術和框架的使用方式
   - 獲得最佳實踐和程式碼範例
   - 了解不同實現方式的優缺點

3. **程式碼品質**
   - 生成符合規範的程式碼格式
   - 提供完整的註解和文件說明
   - 減少語法錯誤和常見 bug

4. **創意激發**
   - 提供多種實現思路，激發創意
   - 快速驗證想法可行性
   - 加速原型開發和概念驗證

### 缺點 ⚠️

1. **需要人工審查**
   - AI 生成的程式碼可能不符合專案架構
   - 需要理解程式碼邏輯並進行調整
   - 必須檢查安全性問題和邊界情況

2. **上下文理解限制**
   - AI 可能無法完全理解專案整體架構
   - 生成的程式碼可能與現有程式碼風格不一致
   - 缺乏對業務邏輯的深度理解

3. **程式碼品質問題**
   - 可能產生冗餘或不優化的程式碼
   - 缺乏對性能考量的深度思考
   - 錯誤處理可能不夠完善
